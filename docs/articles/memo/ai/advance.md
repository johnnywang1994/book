# 進階 AI 使用指南


## 自動化流程
讓 AI 參與到「需求分析 -> 代碼生成 -> 測試驗證」的全流程，甚至是「部署與監控」，是未來的趨勢。這裡的關鍵是「閉環反饋」：AI 不只是執行指令，而是能夠根據測試結果和用戶反饋不斷優化自己的行為。

## 從「執行指令」轉向「主動預警與優化」
目前的流程是「接收指令 -> 執行」。最強的防禦是讓 Agent 具備商業與架構的批判性。
- 加入「挑戰環節」：在建立代辦事項前，增加一個步驟叫「需求合理性審查」。讓 Agent 根據現有代碼庫分析：「這個新功能會不會導致技術債？」或「這是否符合當前的 UI 組件規範？」
- 性能監控 Agent：不只是過測試，開發完後自動跑 Lighthouse 或打包體積分析，如果效能下降，Agent 必須提出優化方案才能通過。

## 異常處理與「邊緣案例」的直覺
目前的自動化流程最怕遇到「測不出來的 Bug」（例如 race condition 或特定瀏覽器的渲染問題）。
- 壓力測試 Agent：除了功能測試，你的流程裡有沒有加入「混亂猴子（Chaos Engineering）」的概念？讓 Agent 專門思考：`如果網絡斷掉會怎樣？如果用戶快速點擊會怎樣？`
- 人類的最後防線：這就是你「防禦策略」的關鍵——當 AI 說「我修好了，測試也過了」，你依然能一眼看出「這裡邏輯雖對，但對用戶不友善」。這種`對產品的「溫度感」是目前的 Agent 最缺少的`。

## 多個 Agent 協作（Multi-Agent Collaboration）
試著為你的 Agent 加入一個「架構師（Architect）」或「產品經理（PM）」的角色來挑戰你目前的開發指令，讓它們互相審查對方的決策。這樣不只是「執行指令」，而是讓 Agent 之間形成一個「內部討論」的機制，從而提升整體的決策質量。
- Agent A (Product Manager)：負責拆解需求與反覆挑戰開發可行性。
- Agent B (Frontend Developer)：負責寫代碼。
- Agent C (QA/Security)：負責找 Bug 與漏洞。
